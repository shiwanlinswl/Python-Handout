chrome浏览器的使用

知识点：
  • 掌握 chrome浏览器抓包的方法
  • 掌握 chrome浏览器查看js运行过程的方法

我们以人人网登陆为例，学习如何使用谷歌浏览器进行抓包，以及查看js运行的过程

1. 使用chrome浏览器进行抓包

1.1 打开隐身窗口

1.2 chrome中network的更多功能

  1. Perserve log
    • 默认情况下，页面发生跳转之后，之前的请求url地址等信息都会消失，勾选perserve log后之前的请求都会被保留

  1. filter过滤
    • url地址很多的时候，可以在filter中输入部分url地址，对所有的url地址起到一定的过滤效果，具体位置在上面第二幅图中的2的位置
  2. 观察特定种类的请求

    • 上图中的3的位置，有很多选项，默认是选择的all，即会观察到所有种类的请求

    • 在all右边有其他选项，是浏览器根据请求对应的响应内容推测的分类，并不准确：

      • XHR：ajax请求
      • JS：js请求
      • CSS：css请求
      • Img：图片
      • Media：音频、视频
      • Font：字体文件
      • Doc：html
      • WS：websocket协议请求
      • Manifest：浏览器本地缓存
      • Other：其它浏览器无法分类的请求
    • 不要被浏览器Net work中的一堆请求吓到了，这些请求中除了js，css，图片的请求外，其他的请求并没有多少个

1.3 寻找登陆接口（url）

寻找url有两种方法，一种是从网页源代码中寻找，比如寻找action对应的url地址；另一只是通过抓包来寻找url

  1. 以人人网为例右键检查寻找action对的url地址

    • 表单提交的地址，对应的，提交的数据，仅仅需要：

      • 用户名的input标签中，name的值作为键，用户名作为值
      • 密码的input标签中，name的值作为键，密码作为值
      • 其它input标签，name的值作为键，value作为值

  2. 以人人网为例通过抓包寻找登录的url地址

    • 通过抓包可以发现，在这个url地址和请求体中均有参数，比如uniqueTimestamp和rkey以及加密之后的password

    • 这个时候我们可以观察手机版的登录接口，是否也是一样的

  3. 谷歌浏览器利用移动端模式抓包

    • 打开新的空页面，右键检查，点选移动端模式，勾选Perserve log选项

    • 输入url地址访问后，找到登陆页面，输入错误的账号密码点击登陆

    • 通过输入的账号来确定登陆的url地址
    • 我们发现登陆请求的url地址对应的post请求参数中
      • rKey参数是由前一次请求获取的响应内容获取的
      • password参数和我们输入的密码并不一致，是由js代码运行加密生成的
知识点：掌握 chrome浏览器抓包的方法

下一小节，我们将学习如何分析js

2. 使用chrome浏览器查看js运行的过程

对于前面人人网的案例，我们知道了url地址中有部分参数，但是参数是如何生成的呢？

2.1 观察标签所具有的js事件

  1. 右键检查 登录按钮

  2. 点击Event Listeners 查看登录标签绑定的js事件

  3. 点击click查看js代码所在的文件

    • 注意不要勾选Ancestors All，那样将显示当前页面上全部的js事件

  4. 点击查看js代码所在文件中的位置

  5. 点击美化输出按钮后，会以黄色高亮的形式提示js代码所在的位置

2.2 观察js的执行过程

观察js的执行过程最简单的方式是添加断点

  • 添加断点的方式：在左边行号点击即可添加，对应的右边BreakPoints中会出现现有的所有断点

  • 添加断点之后继续点击登录，每次程序在断点位置都会停止，通过如果该行有变量产生，都会把变量的结果展示在Scoope中

  • 在上图的右上角有1，2，3三个功能，分别表示：

    • 1：继续执行到下一个断点
    • 2：进入调用的函数中
    • 3：从调用的函数中跳出来

找到js的位置之后，我们可以来通过观察js的位置，找到js具体在如何执行，后续我们可以通过python程序来模拟js的执行，或者是使用类似js2py直接把js代码转化为python程序去执行

知识点：掌握 chrome浏览器查看js运行过程的方法

3. chrome浏览器的其它使用方法：全局搜索

浏览器的全局搜索功能有些时候也可以帮助我们找到js代码或url地址

  • 部分版本全局搜索是Search all file，而有些版本则是Search

找到js的位置之后，我们可以来通过观察js的位置，找到js具体在如何执行，后续我们可以通过python程序来模拟js的执行，或者是使用类似js2py模块直接把js代码转化为python程序去执行，最后获取执行的结果
