功能测试-UI

我们拿到iwebshop电商网站，打开后，首先能看到的就是它的界面，也就是UI，所以先针对UI界面进行测试。

课程目标

  • 熟练SQL语句的编写并转换成测试用例

  • 掌握对于页面初始化内容的测试

UI考虑的点

跟原型图进行比对；
用户的使用习惯；
界面整体测试；
页面内容等。

页面的初始化数据验证

举例：iwebshop前台首页

判断一下数据来自于哪里（数据库）

  • 首先要看一下映入眼帘的首页上的数据对不对。

  • 看一下页面上有哪些内容，接下来进入到数据库中编写sql语句，通过sql语句查询得到一个内容，如果两个内容一样，就证明首页的数据是一致的，如果不一致就证明有bug

  • 拿到一个项目后先查看一下哪些数据是来自数据库。

步骤：

1.先查看首页页面内容，确定页面中与数据库相关的数据(根据有规律排列来判断)

我们可以发现首页中的商品分类、热门搜索、shop资讯、最新商品、公告通知等等都是来自于数据库的。

2.找到各模块相应的数据库中表 

可通过查看数据库文档进行对照（后台-工具-数据库备份中有数据库数据表的列表）

  • 打开iwebshop后台工具模块，有对应的数据库表和相对简单的描述

  • 打开Navicat 连接到数据库

  • 打开对应的iwebshop数据表,查找对应的表和内容

      •商品分类对应的数据库表为iwebshop_category表  
  •热门搜索对应的数据库表为iwebshop_keyword表  
  •shop咨询对应的数据库表为iwebshop_article表  
  •最新商品对应的数据库表为iwebshop_goods表和iwebshop_commend_goods表  
  •热卖商品对应的数据库表iwebshop_goods表和iwebshop_commend_goods表  
  •公告通知对应的数据库表iwebshop_announcement  
  •服饰鞋帽对应的数据库表为iwebshop_category表+iwebshop_goods表+iwebshop_goods_extend表  
  •化妆品 对应的数据库表为iwebshop_category表+iwebshop_goods表+iwebshop_goods_extend表  
  •家居家装对应的数据库表为iwebshop_category表+iwebshop_goods表+iwebshop_goods_extend表  
  •电脑、手机、数码对应的数据库表为iwebshop_category表+iwebshop_goods表+iwebshop_goods_extend表  
  •图书、音像对应的数据库表为iwebshop_category表+iwebshop_goods表+iwebshop_goods_extend表  
  •品牌列表对应的数据库表为iwebshop_brand表  
  •评论表 对应的数据库表为iwebshop_comment表  
  •关键词表iwebshop_keyword表  
  •帮助信息对应的数据库表为iwebshop_help表和iwebshop_help_category

3.根据需求文档编写SQL语句并转换为测试用例 

编写SQL语句时，遇到关键字加``

  • 全部商品分类，iwebshop_category产品分类表

      SELECT name from iwebshop_category where parent_id=0 ORDER BY sort

  • 全部商品分类下的二级分类，iwebshop_category产品分类表

    1. 服饰鞋帽的二级分类 

       select name from iwebshop_category where parent_id=57 and visibility=1 order by id desc

    2. 化妆品的二级分类 

       select name from iwebshop_category where parent_id=67 and visibility=1 order by id desc

    3. 家居、家装的二级分类 

       select name from iwebshop_category where parent_id=68 and visibility=1

    4. 电脑、手机、数码的二级分类 

       select name from iwebshop_category where parent_id=66 and visibility=1 order by id desc

    5. 图书、音像的二级分类 

       select name from iwebshop_category where parent_id=69 and visibility=1 order by id desc

    6. 服饰鞋帽下女装的三级分类 

       select name from iwebshop_category where parent_id=59 order by id

  • 热门搜索，iwebshop_keyword关键词表 

      select word from iwebshop_keyword where hot=1 order by `order` LIMIT 5

  • shop资讯，iwebshop_article文章表 

      SELECT title from iwebshop_article ORDER BY create_time DESC

  • 最新商品，iwebshop_goods+iwebshop_command_goods

      通过在iwebshop_commend_goods中通过推荐类型找到商品id，再拿商品id去iwebshop_goods表找对应商品    
  SQL语句：
      select g.name,g.sell_price,g.market_price,g.img
      from iwebshop_commend_goods as cd 
      inner join iwebshop_goods as g 
      on cd.goods_id=g.id 
      where cd.commend_id=1 
      order by g.up_time desc LIMIT 8

  • 热卖商品，iwebshop_goods+iwebshop_command_goods 

      SELECT `name`,sell_price from iwebshop_goods where id in
  (SELECT goods_id FROM iwebshop_commend_goods where commend_id=3)
  ORDER BY up_time desc
  limit 8

  • 服饰鞋帽，iwebshop_goods+iwebshop_category+iwebshop_category_extend

      先拿iwebshop_category表和iwebshop_category_extend找到商品Id,
  再与iwebshop_goods通过商品Id建立关联

    1.服饰鞋帽id

      select id from iwebshop_category where name ='服饰鞋帽';

    2.服饰鞋帽下的所有二级分类id

      select id from iwebshop_category where parent_id=57;

    3.服饰鞋帽下的所有三级分类id

      select id from iwebshop_category where parent_id in(
  select id from iwebshop_category where parent_id=57
  );

    4.服饰鞋帽所有分类对应的分类id

      select id from iwebshop_category where id in
  (select id from iwebshop_category where parent_id =57)
  or parent_id =57
  or id =57;

    5.根据id查询首页服饰鞋帽显示的商品信息

      select g.name,g.sell_price,g.market_price
  from iwebshop_category as ic
  inner join iwebshop_category_extend as ice
  on ic.id = ice.category_id
  inner join iwebshop_goods as g
  on ice.goods_id=g.id
  where ic.id in(
  select id from iwebshop_category 
  where parent_id in(
  select id 
  from iwebshop_category
  where parent_id =57)
  or parent_id =57)
  order by g.sort,g.id desc
  limit 8;
